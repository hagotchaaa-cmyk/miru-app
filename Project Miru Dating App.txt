How Miru is built and runs:
Stack:

Pure HTML/CSS/JS — no React, no build step, no npm for the frontend
Supabase for database, auth, storage, and edge functions
Stripe for payments (live mode)
Render for hosting (static site, auto-deploys from GitHub main branch)

How deployment works:

All pages are plain .html files in the root of the repo
Push to GitHub main → Render auto-deploys in ~60 seconds
No build command — Render just serves the static files directly
config.js is in the repo root and holds SUPABASE_URL and SUPABASE_ANON_KEY (never committed to GitHub — kept local only)

How auth works:

Supabase Auth handles email/password and Google OAuth
After login, Supabase sets a session cookie — all pages check sb.auth.getUser() on load
Google OAuth redirect goes to https://miru-app.onrender.com/index.html
auth.html is the login/signup page

How payments work:

User clicks Subscribe on premium.html
Page calls Supabase Edge Function create-checkout-session
Edge function creates a Stripe Checkout session using live keys from Supabase Secrets
User completes payment on Stripe hosted page
Stripe fires webhook to stripe-webhook edge function
Webhook sets is_premium = true and saves stripe_customer_id on the user's profile in Supabase

How posts work:

Users upload photos/videos from profile.html via the + FAB button
Files go to Supabase Storage posts bucket
Post metadata (media_url, caption, user_id) saved to user_posts table
discover.html reads from user_posts and shows everyone's posts in a grid

How matching works:

explore.html shows profiles from the profiles table
Swiping right or clicking Connect inserts a row into likes table
If both users liked each other, a row is created in conversations table
messages.html reads from conversations and messages tables in real time

Key files:

config.js — Supabase URL and anon key (local only, gitignored)
All pages are self-contained HTML files with inline CSS and JS
No shared JS files or components — each page is independent

Sensitive keys location:

Stripe keys → Supabase Dashboard → Edge Functions → Secrets
Google OAuth credentials → Supabase Dashboard → Authentication → Providers → Google
Supabase URL/key → local config.js only

Database schema key details:
profiles table columns:

user_id, name, age, bio, avatar_url, banner_url, is_premium, boost_until, stripe_customer_id, stripe_subscription_id, last_shown_at, show_count, created_at

likes table: user_id, target_user_id, type (like/super)
conversations table: user_a, user_b, last_message, last_message_at
messages table: conversation_id, sender_id, content, media_url, media_type
user_posts table: user_id, media_url, media_type, caption, is_approved, flagged
post_likes table: post_id, user_id
post_comments table: post_id, user_id, content
matchmaker_preferences table: user_id, ai_enabled, goal, vibes, delivery_mode, week_mode, age_range, distance_miles

Storage buckets:

avatars — profile photos AND banner photos (banner saved at path: userId/banner/...)
posts — user post media
message_media — photos/videos sent in messages
media — general media

All buckets are public, 500MB limit, no mime type restrictions (null)
